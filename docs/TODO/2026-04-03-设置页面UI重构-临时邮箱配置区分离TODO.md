## 问题描述

当前设置页面 `index.html` 的**临时邮箱配置区**，把两套不同系统的设置混在一起显示：

| 当前展示位置 | 配置项 | 实际归属 |
|---|---|---|
| 统一区块 | 临时邮箱 Provider 选择下拉框 | 全局（选择使用哪套） |
| 统一区块 | 临时邮箱 API 地址 | **GPTMail 专用** |
| 统一区块 | 临时邮箱 API Key | **GPTMail 专用** |
| 统一区块 | 临时邮箱可用域名（JSON） | **混用**（逻辑上两套都读这里） |
| 统一区块 | 临时邮箱默认域名 | **混用** |
| 统一区块 | 前缀规则（JSON） | **混用** |
| 独立区块 | CF Worker 部署地址 | **CF Worker 专用** |
| 独立区块 | CF Worker Admin 密码 | **CF Worker 专用** |
| 独立区块 | 从 CF Worker 同步域名 按钮 | **CF Worker 专用** |

**问题：**
- 用户选了 CF Worker Provider 时，还要去填 GPTMail 的 API 地址，令人困惑
- "临时邮箱可用域名"字段被两套系统共享，但语义不一样（GPTMail 是配置可用域名白名单；CF Worker 是从远端同步过来的只读列表）
- 下拉框式的 Provider 切换不直观，不知道切换后哪些字段会生效

---

## 重构目标

### 方案：改为 Tab 或双面板分离显示

**UI 结构草图**：
```
临时邮箱配置
├── Provider 选择：[• GPTMail  ○ CF Worker]  ← 单选按钮，不是下拉框
│
├── [GPTMail 配置区]（仅当选 GPTMail 时可见）
│   ├── API 地址
│   ├── API Key
│   ├── 可用域名（JSON）
│   ├── 默认域名
│   └── 前缀规则
│
└── [CF Worker 配置区]（仅当选 CF Worker 时可见）
    ├── CF Worker 部署地址
    ├── CF Worker Admin 密码
    ├── [从 CF Worker 同步域名] 按钮 → 自动填充下面两个字段
    ├── 已同步域名（只读展示或可编辑）
    ├── 默认域名
    └── 前缀规则
```

### 关键改进点

1. **Provider 切换改为单选按钮**，不同 Provider 的设置动态显示/隐藏
2. **CF Worker 的域名字段独立**，不与 GPTMail 共用同一个 key（或者至少在 UI 层面分离展示）
3. **CF Worker 域名字段**在点击同步后只读展示（从 CF Worker 自动获取），不允许手动乱填
4. **GPTMail 配置区**在不使用时折叠隐藏，减少视觉干扰
